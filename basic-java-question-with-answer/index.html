Basic Java: Question with answer

June 19, 2020 – by Nitin Kumar 1

What is Java? 

Java is General Purpose ( used in developing a wide variety of applications of different domain), Object-Oriented, Platform independent (write once and run anywhere), Concurrent (multiple work at same time) , very fast and secure programming language. It has automatic memory management (GC), a rich API library, and is free for developers. James Gosling is the father of java.

Java Software family has three members, namely, Java SE, Java EE, Java ME.

What is low-level and high-level language?

Low-level languages are
  1. Machine Language (language that a machine understands) is a set of instructions. CPUs understand these instructions. Any program is a set of instructions. Each instruction has a binary format (0s and 1s). 
  2. Assembly Language is a language much easier to write than the machine code. (eg: add $s0 $s1 ). Used to write instructions. An assembler translates assembly language into machine language.

High-level language Language:  Developers felt the need to create a language much easier to write and to hide low-level details(eg memory allocations). They use English like words. Eg: C, C++, Java.

In High-level language Compiler converts source code to an intermediate code such as machine code(in c) / byte code(in java). Execution happens directly by CPU in a separate stack.  There are few compilers that convert one language(sourceCode) to another, called transcompilers (eg: ClosureScript to JavaScript).

What are the tasks of a Compiler? 

  1.  Verify Syntax (eg statements end with ; ) and semantics (final class is not extended)
  2.  Code Optimisation 
  3.  Generate Machine code.

The interpreter converts source code directly into results. It simulates a CPU. Similar to CPU, it works in fetch-and-execute cycle.

Explain compilation and interpretation in Java.

Java uses both compilation( for fast execution) & interpretation (to achieve platform independence). The compiler first converts source code to byte code (which is not machine code but an intermediate code) and then the interpreter interprets the byte code to get the desired result. The Java interpreter is JVM. JIT (Just-in-time) compilation happens at runtime by JVM which helps in running the program faster. Languages like Groovy & Scala also run on JVM.  

JVM is an abstract computing machine. Like CPU (which reads instruction as machine code and executes it) it also reads instruction set as byte code. It also manipulates memory at runtime.

How does a java program run ?

Let’s say we have a program called ‘Hello.class’. When we run the command ‘java Hello’, an instance of JVM is created and loaded into memory. JVM then in turn loads ‘Hello.class’ program into memory and executes it. One JVM instance runs only one java application.

When JVM is created, a chunk of memory (called runtime data areas or JVM memory) is obtained from the underlying operating system. In order to load the ‘Hello.class’ , the JVM invokes a component called class loader. The class loader’s responsibility is to locate the Hello.class and load the corresponding byte code into memory. JVM Memory has two important areas: heap and method area. Byte code once loaded is verified by a component called the bytecode verifier. Bytecode verifier verifies whether the loaded byte code is not corrupted or tampered by hackers. Once bytecode verification passes, the bytecode is passed to the execution engine (which includes JIT compiler and interpreter). This interpreter helps in achieving platform independence and JIT compiler help in very fast execution speed. Along with this, we have a component called Garbage collector (GC) which has the responsibility of managing the memory of the JVM. GC reclaims the memory no longer in use. There is a component called security manager, which ensures security, one of the core responsibilities of JVM. The security manager can help in running code in a more restricted environment (like preventing code to access file systems) called Sandbox.

What is JDK and JRE?

JDK means Java Development Kit which includes development Tools, JVM, Java API.

There are many implementations of JDK in the market. (eg Oracle JDK, Oracle OpenJDK, Red Hat’s Open JDK, etc) . All the implementation originated from a project called Open JDK, originally owned by Oracle but later made open source so that anyone can contribute. The source code of OpenJDK includes API and JVM called HotSpot.
JRE includes JVM, Java API. Separate JRE was offered only till Java SE 10. After SE 10, oracle started to provide a tool called jlink which can be used to create a custom JRE. This custom JRE might be smaller in size than regular JRE. Instead of creating custom JRE, one can always use JDK as it always has JRE.

How is Java platform independent?

Bytecode generated by the compiler is common across all devices. JVM has Interpreter which converts bytecode into machine level code which varies from OS to OS. Note that Interpreter is different across platforms.

Explain object creation process in java

Whenever a new class instance is created, memory space is allocated for it with room for all the instance variables declared in the class type and all the instance variables declared in each superclass of the class type, including all the instance variables that may be hidden. If there is not sufficient space available to allocate memory for the object, then the creation of the class instance completes abruptly with an OutOfMemoryError (does not throw null). Otherwise, all the instance variables in the new object, including those declared in superclasses, are initialized to their default values.

What is the difference between Virtual Machine and Sandboxed Environment?

A virtual machine provides a simulated computer that guest applications will see as the whole system. It is not necessarily sandboxed since it can provide write access to any shared location on the host computer.

A sandboxed environment is not necessarily virtualized: it does not have to simulate a computer, just restrict access permissions for running applications. Javascript within your browser provides such an environment without virtualization.
