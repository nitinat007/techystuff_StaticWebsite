What are the common interfaces and their implementations in the Collection framework of Java? 

  • List interface is implemented by the classes ArrayList, LinkedList, Vector, and Stack.
  • Vector, Stack are synchronized. ArrayList, LinkedList are non-synchronized
  • Queue interface is implemented by PriorityQueue, Deque (doubly ended queue)
  • Set is implemented by HashSet, LinkedHashSet, and TreeSet
  • LinkedHashSet class represents the LinkedList implementation of Set Interface. It maintains the insertion order and permits null elements

HashSet vs TreeSet. Which one should I use and why?

HashSet : Better time complexity. No order is maintained, Can store null value

TreeSet : Maintains order in ascending order, Can’t store null value

Common thing : No duplicate, non synchronized, 

What is the difference between HashMap, Hashtable and SynchronizedMap ?

All of them use Map data structure to keep the data but one main difference is locking.

1.HashMap

  • HashMap is an implementation of Map data structure
  • for single threaded environment
  • Not thread-safe
  • can keep one null key and multiple null value.

2. HashTable

  • Hashtable is an implementation of Map data structure
  • This is a legacy class in which all methods are synchronized on Hashtable instances using synchronized keyword.
  • Thread-safe as it’s method are synchronized

3. SynchronizedMap

  • This is wrapper of Map, and can be initialized by using Collections.synchcronizedMap(new HashMap())
  • similar to Hashtable
  • The only difference between Hashtable and Synchronized Map is that later is not a legacy and you can wrap any Map to create it’s synchronized version by using Collections.synchronizedMap() method.

What is  ConcurrentHashMap? Why should you prefer using it over Hashtable? (introduced in java 1.5)

  • ConcurrentHashMap implements Map data structure and also provides thread safety like Hashtable.
  • It works by dividing the complete hashtable array into segments or portions and allowing parallel access to those segments.
  • The locking is at a much finer granularity at a hashmap bucket level.
  • Reads can happen very fast while writes are done with a lock on segment level or bucket level.
  • Use ConcurrentHashMap when you need very high concurrency in the application.
  • It is thread-safe without synchronizing the whole map. There is no locking at the object level.
  • ConcurrentHashMap doesn’t throw a ConcurrentModificationException if one thread tries to modify it while another is iterating over it.
  • ConcurrentHashMap does not allow NULL values, so key can not be null in ConcurrentHashMap
  • ConcurrentHashMap doesn’t throw a ConcurrentModificationException if one thread tries to modify it, while another is iterating over it. ConcurrentHashMap uses fail safe iterator (not fail fast iterator). But if a hashtable is structurally modified during iteration, concurrentModificationException occurs.  ConcurrentHashMap makes a copy of the internal data structure (object array) and iterates over the copied data structure.
  • Ways to initialize ConcurrentHashMap:

ConcurrentHashMap premiumPhone = new ConcurrentHashMap();

ConcurrentHashMap premiumPhone  = new ConcurrentHashMap (int initialCapacity, float loadFactor, int concurrencyLevel);
