Variables & Methods: Question with Answer

July 9, 2020 – by Admin

Explain different categories of variables in Java

Variables are of following three categories
  1. Instance variable: They are directly under the class body and store the state of the object
  2. Static Variable: They are directly under the class body but also have a static keyword. They are shared between different objects of class.
  3. Local Variable: These variables are declared within a method. This must be initialized before use.

Variables are also categorized as 

  1. Primitive: byte(8 bit), short(16 bit), int(32 bit), long(64 bit), float(32 bit), double(64 bit), char(16 bit unsigned integer with range from 0 to [2^16]-1), boolean(1 bit)
  2. Reference Variable: Array, ArrayList, Variables of any instantiable class.

Explain casting variables in java

  1. Implicit: Happens from variable of smaller to larger size. This is done by the compiler automatically. Examples are: When an int variable is assigned to a long variable. When a char type variable is assigned to int or long or float or double.
  2. Explicit: Programmer has to cast variables of larger size to smaller size. There is information loss in the process. Eg of explicit casting:

long a=22;

int b=(int)a;

What is type checking in java?

Type checking is the process of analysing a program to ensure that the types of expressions are consistent.Type checking happens in a programming language to prevent error. In Java type checking happens at both compile time as well as runtime. Type checking at compile time is called static type checking. Type checking at runtime (eg ArrayIndexOutOfBoundsException, ClassCastException) is called dynamic type checking. Eg

int a = 22.22;  // Throws compile time error. Do Implicit cast to prevent this

   user.doAdminTask();      //Do Implicit cast to prevent compilation error but this may cause runtime time error (ClassCastException) if user is not an admin User. 

Generics (java 5) was introduced to catch as many errors as possible at compile time itself. Java is a type safe language. C does only static-type checking. Python does only dynamic-type checking.

What is method binding in java?

Method binding means binding a method to method declaration and its implementation. Compiler checks if the reference type has a compatible method. If found then it writes the method signature into the byte code it generates. If not found it throws a compiler error. This is always done by the compiler, irrespective of the methods being static or non static(instance type) . This process is called method – signature binding. If the method is static, then the compiler decides the method to be invoked (based on reference type). But if the method is of instance type (non static) then the JVM decides the method to be invoked (based on the object type) at runtime. So, for static methods the method binding happens at compile time itself (early binding) . But binding of methods of instance type has to wait till the runtime as it is dependent on object type (late binding). At runtime if the method is found at the object’s reference, the JVM invokes it else it tries to find the methods one step above in the inheritance tree. Polymorphism and method overriding follows late binding.

Instance methods are late bounded but instance fields (including static) are early bounded.

Can we override a static method in Java? 

No. You cannot override a static method because it’s not bound to an object. Instead, static methods belong to a class and are resolved at compile time using the type of reference variable. But, Yes, you can declare the same static method in a subclass, that will result in a method hiding i.e. if you use reference variable of type subclass then the new method will be called, but if you use a reference variable of superclass then old method will be called.

Can we prevent overriding a method without using the final modifier? 

Yes, you can prevent the method overriding in Java without using the final modifier. You can mark the method private or static, those cannot be overridden. In fact, there are several ways to accomplish this.

Can we change the return type of method to subclass while overriding? 

Yes, we can do this from Java 5 onward. This feature is known as the covariant method overriding.

What is Covariant return type in Java?
Since Java 5, it is possible to override a method by changing its return type. If a subclass overrides any method by changing the return type of the super class method, then the return type of the overridden method must be a subtype of return type declared in the original method inside the super class. This is the only way by which method can be overridden by changing its return type.

Here the Super-class method returns a superclass type Object and the subclass method returns a subtype of the superclass method’s return type. By using this, you can directly return the actual type, preventing client-side type casting of the result.  e.g. clone() method return java.lang.Object 

Can we override a method which throws a runtime exception without adding throws clause in method declaration? 

Yes, there is no restriction on unchecked exceptions while overriding. On the other hand, in the case of checked exception, an overriding exception cannot throw a checked exception which comes higher in type hierarchy e.g. if original method is throwing IOException then overriding method cannot throw java.lang.Exception or java.lang.Throwable.

Can we override a non-static method as static in Java? 

Yes, you can override the non-static method in Java. But it should not be private or final 

Can we overload or override the main method in Java? 

No, since main() is a static method, you can only overload it, you cannot override it because the static method is resolved at compile time without needing object information hence we cannot override the main method in Java

Is it good practice to concatenate String using + operator?

No. Beware of the performance of + operator on String. Everytime you concatenate using + , the content of each string is copied, a new StringBuilder is created and appended and finally returned by calling toString() method on StringBuilder.  It is time consuming O(n2) and space consuming as several string and stringBuilder references are created in the process. Concatenation(append) using StringBuilder is 300X faster than concatenation using + operator in String and 2X faster than StringBuffer. So, use + operator only to combine a few strings.

What is the significance of toString() method on object?

The toString() method returns the string representation of the object.

Whenever we try to print any object of a class,  its toString() method will be called. If we don’t override the toString() method in the class and call the toString() method, then it would print the object id.

What is StringBuffer? How is it different from String Class?

StringBuffer (mutable) class is used when we have to make a lot of modifications to our string. It is also thread safe i.e multiple threads cannot access it simultaneously.

String class is immutable.

  StringBuffer strB = new StringBuffer(“hello”);
  strB.append(“wor”).append(“ld”);
  System.out.println(strB);    // prints: helloworld
