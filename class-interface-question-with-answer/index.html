Class & Interface: Question with Answer

July 9, 2020 – by Admin

What is the difference between an abstract class and an Interface in Java?

Below are the differences between an abstract class and an Interface

  1. You can not extend more than one class because Java does not support multiple inheritances but you can implement multiple inheritance in Java.
  1.  Interface is better suited for Type declaration and abstract class is more suited for code reuse and evolution perspective.
  2.  Abstract classes are slightly faster than interfaces because interface involves a search before calling any overridden method in Java.
  1. When you add a new method in an existing interface it breaks all its implementation and you need to provide an implementation in all clients. By using an abstract class you can provide a default implementation for a new method in the superclass without breaking existing clients. Java 8 onwards we can have default implementation of methods in an interface.

Why was the default implementation of interface introduced in Java 8 ?

Default implementation of an interface allows us to add new methods to an interface that are automatically available in the implementations. Thus, there’s no need to modify the implementing classes. In this way, backward compatibility is neatly preserved without having to refactor the implementations. With java 8 interfaces can contain implementation of default methods and static methods. Default method must have a keyword default. 

When to use an interface and when to use an abstract class?

When you know partial behavior of a class you can have a predefined method for known behavior and for declare methods whose behavior is unknown/may vary as per the implementation in an abstract method. Eg. HttpServlet is an abstract class. Methods for creation and destruction are predefined and developers have to write their own implementation for certain other methods (like service(ServletRequest req,ServletResponse res) ).

Interface is created when you know only the contract and you do not know the behavior at the time of creation. Any class that implements this interface has to write behaviour of each and every contract.  Eg: Map is interface , HashMap , HashTable , TreeMap , WeekHashMap are implementation of the interface. 

In General prefer interfaces to abstract classes. Additional capability (mixins) can be added to a class by implementing an interface. Even when a class already extends a class (or already implements another interface) we can still implement this mixin interface. This would not be possible with abstract class. 

Prefer abstract class over interface if you have to define core identity of subclass. I.e you know the abstract behavior (identity) of sub classes and state. Programmer tries to give skeletal implementation using abstract class. Eg: Animal class with variables defined (eg name, makeNoise,isWild etc) should be an abstract class rather than an interface.

Can we have a main() method in an interface ?

Yes, We can have a psvm (public static void main) method in an interface from Java 8 onwards. Because static methods are allowed in an interface starting from Java 8. 

Interfaces don’t allow the use of static, until Java 7.

But of course, you cannot override the main method, since psvm is a static method.

What is the difference between Composition and Inheritance in OOP? Though both Composition and Inheritance allows you to reuse code, formerly is more flexible than later. , Composition allows a class to be composed of multiple parts whereas inheritance allows a class to be derived from another class. Composition allows the class to get an additional feature at runtime

Explain the Externalizable interface in Java

The Serializable interface gets us automatic serialization capability for objects of our class. On the other hand the Externalizable interface provides a way to implement a custom serialization mechanism. A class that implements the Externalizable interface is responsible to save and restore the contents of its own instances.

The Externalizable interface extends the Serializable interface and provides two methods to serialize and deserialize an object, writeExternal() and readExternal().

What is the Marker interface in Java?

There are few interfaces which do not have any method or field. Marker interface is used to inform the compiler that the class implementing it has some special behaviour or meaning.  However marker interfaces have been deprecated since Java 5, they were replaced by Annotations. eg: 

java.io.serializable
java.lang.Cloneable → provide ability to clone the object.
java.rmi.Remote
java.util.RandomAccess → provides utility for accessing random element faster

What is Nested Class ?  

Nested class is a class with another class. Example : HashMap class in collections has a static class Node within it, with method getKey(). 

Nested Class is of two type : 

  1. Inner Class
  2. Static Member Class

What is Inner Class?

If you want to create a class which is to be used only by the enclosing class, then it is not necessary to create a separate class file for that. Instead, you can add it as “Inner Class”, i.e class within class.

If a class named Outer has a class named Inner within it, this is how you instantiate the inner class

  Outer ot = new Outer();
Outer.Inner innerObject = ot.new Inner();

Accessing methods ( Here Outer is the outer class name and Inner is the inner class name)

  1. Outer class can access inner class method by: innerObject.innerMethod()
  2. Inner class can access the outer class method by: outerMethod(). Here outerMethod of Outer class is invoked only if Inner class does not have the same method.
  3. From the inner class this.go() will always refer to the inner class’s go() method.
  4. From the inner class Outer.this.go() will always refer to the Outer class’s go() method.

What is Static Member Class?

A static class created inside a class is called a static member class. It can access only static members (variables, methods including private) of outer class. We should favour static member class over Inner class. 

If a class named Outer has a static class named Inner within it, this is how you instantiate the inner class

 Outer.Inner obj=new Outer.Inner();  

Accessing static field: Outer.StaticClass.staticVariable

When do you use Inner Class Vs Static Member Class?

Static Member Class can not access instance members of enclosing class whereas Inner Class can access instance members of enclosing class. If the class within the enclosing class needs access to the outer (enclosing) class, we declare Inner class than static member class.

What are the key points to note about nested class? How is it different from (Outer) Class?

  1. Nested classes can have all access levels whereas top-level (Outer) class can have only public or default access. 
  2. Outer class can access any member (including private) of nested class and also nested class can access any member of outer class. 
  3. For outer and inner class named Outer and Nested, compiler generates .class file named Outer.class and Outer$Nested.class
  4. You can not have a static member within Inner class. This will throw a compile time error as in order to get access to the inner class you need to create an instance of outer class (say outerObject)  and then access the static member by outerObject.memberName. It does not make sense to access it by objectName.memberName for a static member.

What is an Anonymous class in Java?

It is a class that does not have a name.  

Example of anonymous class: Here is an example of creating a thread where the class implementing Runnable interface has no name.

Thread t = new Thread(new Runnable() {

   @Override

   public void run() {

       // something

   }

});

What is a Local class?

These are classes that are defined within a method. We cannot use private, public, or protected access modifiers with local inner class. Only abstract and final modifiers are allowed. This class is mostly declared when the scope of a class is only within the method. They are rarely used. 

What are Access Modifiers? 

Access Modifiers: public, private, default , protected. 

What is the difference between default and protected modifiers?

Default: Scope is within a package

Protected: method defined as protected can be accessed within a package and any class in another package which is inheriting the class with protected method. 

Class can’t have a protected modifier (only method can have).

What are Non-Access Modifiers? 

Non-Access Modifiers: static, abstract, synchronized, native, volatile, transient, etc.
