What is an encoding scheme?

Every file uses some encoding schemes to map its character. Encoding schemes is an algorithm which maps character to hexadecimal numbers and whose binary representation is used. Eg

Encoding:  ‘a’ → 61(hexadecimal) → 01100001(binary)

Decoding: 01100001 → 61 → ‘a’

Encoding Schemes example : ASCII, UTF-16, UTF-32. More than half of web pages are encoded in UTF-8

What are Streams? What are the different types of Streams?

Java performs I/O through Streams.

Types of streams:

1) byte stream: for IO of bytes (handling binary data eg images,audio,video)

2) character stream: for IO of characters (for handling character data eg text files)

What are different classes for IO in Java? 

OR

Explain InputStream, OutputStream, Reader, Writer and classes extending these Classes.

InputStream is an abstract class with overloaded read() abstract methods and is used for reading Byte Stream. OutputStream is an abstract class with overloaded write() abstract methods and is used for writing Byte Stream. Reader is an abstract class for reading Character Stream. Writer is an abstract calls for writing Character Stream. 

InputStreamReader can also be used for reading character streams. Both InputStream & OutputStream read/write data in 8-bit bytes.

Classes extending abstract InputStream: FileInputStream, ObjectInputStream, ByteArrayInputStream etc

Classes extending abstract OutputStream: FileOutputStream, ObjectOutputStream, PrintStream etc

Classes extending abstract Reader: BufferedReader,FileReader, StringReader etc

Classes extending abstract Writer: BufferedWriter,FileWriter, StringWriter, PrintWriter etc

It is recommended to use FileReader for character Stream and InputStreamReader for byte stream.

View Answer

Why do we use BufferedInputStream & BufferedOutputStream  in Java? Also 

Reading and writing using byte by byte is way too expensive as it needs disk access. We use buffering to overcome this. Here we read/write a block of bytes into a memory buffer (byte array) and finally read/write (flush) from this buffer in a single IO operation. Default size of the memory buffer is 8192 byte,which can be changed by a programmer. We have BufferedInputStream & BufferedOutputStream for this.  These streams do not work independently. They work with other streams like FileInput/FileOutput streams. They have buffering as the main functionality and do not deal with reading and writing. Eg

BufferedInputStream bis = new BufferedInputStream(new FileInputStream(“go.jpg”));

What is the design Pattern used for Reading and Writing using BufferedInputStream, BufferedReader etc? 

This design pattern is called a decorator design pattern as one object is wrapping another object. Both BufferedInputStream and FileInputStream have a common super type. Note that it is following the design principle of Open-For-Extension-Close-For-Modification.

bis.read() method will additionally add buffering functionality to fis.read().

Why do we club BufferedReader with InputStreamReader for reading in Java?

BufferedReader reads a couple of characters from the specified stream and stores it in a buffer. This makes input faster. InputStreamReader reads only one character from a specified stream and remaining characters still remain in the stream.

What is Serialisation and Deserialization in Java?

Serialization is a process of converting an object into a sequence of bytes which can be persisted to a disk or database or can be sent through streams.  ObjectOutputStream connected to a FileOutputStream. Deserialization is the reverse process where the byte stream is used to recreate the actual Java object in memory. Deserialization does not call the constructor of the class. But if there is any non serializable ancestor, then the ancestor constructor is run.

How can you prevent a data member from getting serialized?

Transient keyword will prevent that data member from being serialized (deserialized value will be default value of the type).

What happens to the objects that the serializing object references to? Do they also get serialized during serialization?

If an object is serialized then any object it references is also serialized (i.e saves the entire object graph). If super class is serializable then subclass is automatically serializable.
View Answer

How do you deserialize an object whose class has changed/evolved over time?

Deserialization fails if the class for which object was serialized has evolved. Serializable classes get a default serialVersionUID generated for that class based on class structure during serialization. When an object is deserialized this ID is compared b/w object and class. To allow the object deserialization even when class has changed, we need to add static final long serialVersionUID=8883434……8888L; 

Value of serialVersionUID can be generated using a tool serialver provided in JDK which takes a class file location argument.

How can you improve the performance of ObjectOutputStream?

We can also chain ObjectOutputStream with BufferedOutputStream for improving performance, like this

Objectoutputstream oos = new ObjectOutputStream(new BufferedOutputStream(new FileOutputStream(“abc.ser”)));
